## 1. プロジェクト概要

### 1.1 目的
就職活動における企業ごとの選考ステータスや各種締切を一元管理・可視化するWebアプリケーション。
Googleアカウントによるシングルサインオンで、PC・スマートフォンを問わず即座にアクセスでき、リアルタイムにスケジュールを確認・更新できることを主眼とする。

### 1.2 対象ユーザー
- 就職活動中の学生・求職者

### 1.3 利用環境
- Webブラウザ（Chrome / Firefox / Safari / Edge 最新版）
- スマートフォンブラウザ（レスポンシブ対応）

## 2. 機能要件

### 2.1 認証・ユーザー管理

| # | 機能 | 詳細 |
|---|------|------|
| A-01 | Googleログイン | Auth.js（NextAuth.js）+ Google Provider によるOAuth2認証 |
| A-02 | セッション管理 | JWTセッションをFastAPI側でセキュアに検証（署名検証必須） |
| A-03 | 自動ログアウト | セッション有効期限切れ時に自動でログイン画面へリダイレクト |
| A-04 | ユーザーデータ分離 | ログインユーザーのデータのみ参照・編集可能（他ユーザーのデータへのアクセス禁止） |

### 2.2 カンバンボード（選考ステータス管理）

| # | 機能 | 詳細 |
|---|------|------|
| B-01 | カンバン表示 | 選考ステータスをカラム単位で一覧表示 |
| B-02 | ドラッグ＆ドロップ | 企業カードをドラッグして選考ステータスを直感的に変更 |
| B-03 | リアルタイム反映 | ステータス変更をAPIを介して即座にDBへ保存し、UIへ反映（楽観的更新） |
| B-04 | カード作成 | 企業名・業界・志望度・メモを入力して新規エントリーを追加 |
| B-05 | カード編集 | カードクリックでモーダルを開き詳細情報を編集 |
| B-06 | カード削除 | 確認ダイアログを経て企業エントリーを削除 |
| B-07 | カラムカスタマイズ | デフォルトステータス以外に独自ステータスを追加・削除・並び替え可能 |

#### 2.2.1 デフォルト選考ステータス（カラム）

| 順序 | ステータス名 | 説明 |
|------|-------------|------|
| 1 | エントリー | 企業情報を登録した初期状態 |
| 2 | ES提出 | エントリーシート提出済み／提出予定 |
| 3 | 書類選考 | 書類選考結果待ち |
| 4 | 一次面接 | 一次面接予定・結果待ち |
| 5 | 二次面接 | 二次面接予定・結果待ち |
| 6 | 最終面接 | 最終面接予定・結果待ち |
| 7 | 内定 | 内定獲得 |
| 8 | 不合格 | 選考落ち（アーカイブ） |
| 9 | 辞退 | 自己都合辞退（アーカイブ） |

### 2.3 締切・日程管理とカウントダウン

| # | 機能 | 詳細 |
|---|------|------|
| C-01 | 締切登録 | ES提出期限・面接日時・説明会日時など複数の日程を企業カードに紐付け登録 |
| C-02 | カウントダウン表示 | 各締切までの残り日数・残り時間をカードおよび詳細画面にリアルタイム表示 |
| C-03 | 期限切れハイライト | 締切を過ぎた項目を視覚的に強調表示（赤色・アイコン） |
| C-04 | 直近締切サマリー | ダッシュボードに「今週の締切」「明日の予定」を一覧表示 |
| C-05 | リマインダー通知 | 締切X日前・X時間前にブラウザプッシュ通知を送信（Web Push API） |
| C-06 | タイムゾーン対応 | ユーザーのブラウザタイムゾーンに基づき日時を表示 |

### 2.4 Googleカレンダー連携（将来対応・Phase 2）

| # | 機能 | 詳細 |
|---|------|------|
| D-01 | カレンダー同期 | 面接日程・締切をGoogleカレンダーへ自動登録（Google Calendar API v3） |
| D-02 | 双方向同期 | Googleカレンダー側で変更された日程をアプリに反映 |
| D-03 | イベント削除連携 | アプリで日程を削除した際にGoogleカレンダーのイベントも削除 |
| D-04 | OAuth2スコープ追加 | Googleログイン時に `https://www.googleapis.com/auth/calendar` スコープを追加要求 |

> **Note:** Phase 2 対応のため、DBスキーマにはGoogleカレンダーイベントID（`google_event_id`）カラムを初期から設けておく。

### 2.5 ダッシュボード

| # | 機能 | 詳細 |
|---|------|------|
| E-01 | 統計サマリー | エントリー社数・面接予定数・内定数をカード形式で表示 |
| E-02 | 直近スケジュール | 7日以内の締切・面接を時系列リストで表示 |
| E-03 | ステータス分布グラフ | 選考ステータス別の企業数を棒グラフ・円グラフで可視化 |

## 3. 非機能要件

### 3.1 パフォーマンス
| 項目 | 目標値 |
|------|--------|
| 画面初回表示 | 2秒以内（LCP） |
| APIレスポンス | 200ms以内（P95） |
| カンバンドラッグ操作 | 楽観的更新で即時反映（ネットワーク待ちなし） |

### 3.2 セキュリティ
| 項目 | 対策 |
|------|------|
| 認証トークン検証 | FastAPI側でAuth.js発行JWTの署名・有効期限・issuer を検証 |
| CORS設定 | 許可オリジンをフロントエンドのドメインのみに限定 |
| SQLインジェクション対策 | SQLAlchemy ORMのパラメータバインドを使用（生SQLクエリ禁止） |
| XSS対策 | Next.jsのデフォルトエスケープ + dangerouslySetInnerHTML 使用禁止 |
| HTTPS強制 | 本番環境ではHTTPS必須、HSTSヘッダー付与 |
| ユーザーデータ分離 | 全APIエンドポイントでJWTのuser_idとDBのuser_idを照合 |

### 3.3 可用性・運用
| 項目 | 方針 |
|------|------|
| 開発環境 | Docker Compose による全サービス一括起動 |
| DBマイグレーション | Alembic を使用。マイグレーションファイルをGit管理 |
| ログ | FastAPI側でリクエストログ・エラーログを構造化JSON出力 |

### 3.4 レスポンシブデザイン
- モバイルファースト設計（Tailwind CSS のブレークポイントを活用）
- カンバンボードはモバイルでは縦スクロール、PCでは横スクロール

## 5. APIエンドポイント概要

| メソッド | パス | 説明 |
|---------|------|------|
| GET | `/api/companies` | 企業一覧取得（ログインユーザーのみ） |

| GET | `/api/companies/{id}` | 企業詳細取得 |
| PATCH | `/api/companies/{id}` | 企業情報更新（ステータス変更含む） |
| DELETE | `/api/companies/{id}` | 企業削除 |
| GET | `/api/companies/{id}/events` | 企業に紐づく日程一覧取得 |
| POST | `/api/companies/{id}/events` | 日程登録 |
| PATCH | `/api/events/{id}` | 日程更新 |
| DELETE | `/api/events/{id}` | 日程削除 |
| GET | `/api/statuses` | ステータス一覧取得（カラム定義） |
| POST | `/api/statuses` | カスタムステータス追加 |
| PATCH | `/api/statuses/{id}` | ステータス更新・並び順変更 |
| DELETE | `/api/statuses/{id}` | カスタムステータス削除 |

## 6. 制約・前提条件

- ユーザーはGoogleアカウントを保有していること
- 初期リリース（Phase 1）ではGoogleカレンダー連携は対象外とし、DBスキーマのみ対応
- データは個人利用を前提とし、チーム共有機能は対象外
- 企業データのエクスポート機能（CSV等）はPhase 2 以降に検討

## 7. 用語定義

| 用語 | 定義 |
|------|------|
| エントリー / 企業カード | 1社の就活情報をまとめた管理単位 |
| 選考ステータス | 企業カードが属するカンバンのカラム（選考フェーズ） |
| イベント | 締切・面接・説明会などの日程情報（企業カードに紐づく） |
| カウントダウン | イベント日時までの残り時間をリアルタイムに表示する機能 |
